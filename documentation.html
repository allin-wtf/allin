üéØ Plinko Game

Game Mechanics

Plinko is a game of chance inspired by the classic pachinko game. A ball drops from the top, bouncing off pegs, and lands in one of 17 multiplier slots at the bottom.

Board Configuration

  • 16 rows of pegs arranged in triangular pattern
  • 17 multiplier slots at the bottom
  • Binomial distribution determines final position
  • Physics simulation with gravity and collision detection
  • 800px √ó 600px game board dimensions

Probability Distribution

Plinko follows a binomial distribution. Each peg bounce is a 50/50 decision (left or right), creating a bell curve distribution.

Binomial Probability Formula:
P(X = k) = C(n, k) √ó (0.5)^n

Where:
‚Ä¢ n = number of rows (16)
‚Ä¢ k = position (0 to 16)
‚Ä¢ C(n, k) = binomial coefficient = n! / (k! √ó (n-k)!)

Example Calculation: Center Slot (Position 8)

P(X = 8) = C(16, 8) √ó (0.5)^16
P(X = 8) = 12,870 √ó 0.0000152587890625
P(X = 8) = 0.196380615234375
P(X = 8) ‚âà 19.64% or 21.09% (with edge adjustments)

House Edge

The house edge in Plinko is achieved through the multiplier distribution:

Expected Return Calculation

Expected Return = Œ£ (Multiplier √ó Probability)
= (110 √ó 0.0015%) + (41 √ó 0.0244%) + ... + (0.3 √ó 21.094%)
= 98.0%

House Edge = 100% - 98.0% = 2.0%

Over/Under Strategy

Optimal Predictions

For Over/Under bets, prediction value of 10 or 11 provides maximum fairness:

  • Over 10: Sums 11-18 = 120 combinations / 216 = 55.56%
  • Under 11: Sums 3-10 = 120 combinations / 216 = 55.56%
  • Expected Return: 1.8 √ó 0.5556 = 100% (no house edge!)

Randomness Generation

Cryptographic RNG Process

  1. Server generates seed from crypto.randomBytes(32)
  2. Client timestamp provides additional entropy
  3. Combined hash: SHA-256(server_seed + client_seed + nonce)
  4. Hash converted to three numbers (1-6) using modulo operation
  5. Result stored on-chain for verification

‚úÖ Provable Fairness

What is Provable Fairness?

Provable fairness is a cryptographic system that allows players to verify that game outcomes are truly random and not manipulated by the house. Every game result can be independently verified using blockchain data.

How We Ensure Fairness

1. Cryptographic Random Number Generation

  • Server generates random seed using crypto.randomBytes()
  • Client provides timestamp and browser entropy
  • Seeds combined with SHA-256 hashing
  • Result deterministic but unpredictable

2. Blockchain Verification

  • Every game result recorded on Solana blockchain
  • Transaction signature provides cryptographic proof
  • Immutable record prevents tampering
  • Anyone can verify on Solscan

3. Smart Contract Execution

  • Game logic executed in smart contracts
  • Deterministic outcomes based on seeds
  • No human intervention possible
  • Code is open and auditable

Verification Process

Step 1: Server generates server_seed (hidden from player)
Step 2: Server sends hash of server_seed to client
Step 3: Client generates client_seed (timestamp + random)
Step 4: Game played with combined seeds
Step 5: Server reveals server_seed after game
Step 6: Player verifies:
    ‚Ä¢ SHA-256(server_seed) = previously provided hash ‚úì
    ‚Ä¢ Recompute game result with both seeds ‚úì
    ‚Ä¢ Compare with actual outcome ‚úì

Mathematical Proof

RNG Algorithm

combined_seed = SHA-256(server_seed + client_seed + nonce)
random_number = parseInt(combined_seed.substring(0, 8), 16)
result = (random_number mod range) + min

Example for Dice (1-6):
range = 6, min = 1
dice_value = (random_number mod 6) + 1
